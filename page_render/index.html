Overview

BTL is a comprehensive 3D Graphics Engine built with Modern OpenGL (3.3+) and Python, designed for visualizing complex geometries, loss surfaces, and optimization algorithms in real-time. It combines interactive visualization with scientific computing to provide a powerful tool for understanding machine learning optimization landscapes.

✨ Key Features

🎯 Core Capabilities

  • Interactive 3D Viewer - Real-time visualization of 2D/3D geometries using Modern OpenGL
  • Multiple Shading Models - Flat, Gouraud, Phong, and Extended Phong shading algorithms
  • Optimization Visualization - Animate 5 major optimization algorithms (SGD, Adam, Momentum, etc.)
  • Loss Surface Rendering - Visualize 5 loss functions: Beale, Himmelblau, Rosenbrock, Booth, Quadratic
  • Advanced Lighting System - Multiple lights, HDRI environment mapping, custom light sources
  • Real-time Material Control - Dynamically adjust roughness, metallic properties, and emissive colors
  • Dual Camera System - Free orbit camera + fixed angled view with independent controls
  • Contour Maps - Split-view or inset 2D loss surface contours with trajectory overlays

🔧 Technical Features

  • GPU-Accelerated Rendering - Vertex and Fragment shaders with 5 shading configurations
  • Efficient Geometry Management - VAO/VBO/EBO-based buffering system
  • Interactive UI - ImGui control panel for real-time parameter adjustment
  • Modular Architecture - Separated concerns: geometry, rendering, physics simulation, visualization
  • Trajectory Recording - Playback and visualization of optimizer paths
  • Ball Physics Simulation - Rolling ball animation for optimizer visualization with decay and lift

Common Issues & Solutions

Issue 1: ModuleNotFoundError for imgui

# Use conda-forge (has better pre-built binaries)
conda install -c conda-forge imgui

# OR manually build
pip install --upgrade pip setuptools wheel
pip install imgui[glfw]

Issue 2: PyOpenGL Error - "No module named OpenGL"

# Clear and reinstall
pip uninstall PyOpenGL
pip install --force-reinstall PyOpenGL

Issue 3: GLFW Window Fails to Create

macOS:

brew install glfw
pip install glfw

Linux (Ubuntu):

sudo apt-get install libglfw3-dev libglew-dev
pip install --force-reinstall PyOpenGL glfw

Issue 4: OpenGL Version Error

# Check OpenGL version
python3 -c "
from OpenGL.GL import glGetString, GL_VERSION
import OpenGL.GL as GL
print(glGetString(GL_VERSION))
"

Update GPU drivers if necessary

Issue 5: ImGui Freezes or Doesn't Render

# Try alternative ImGui backend
pip uninstall imgui
pip install imgui[glfw] --force-reinstall

Issue 6: Performance Issues (Low FPS)

  • Close other GPU-intensive apps
  • Lower "Grid resolution" slider from 200 to 100
  • Disable HDRI environment mapping
  • Reduce optimizer count

Example 5: Compare All Optimizers

from BTL.optimizer_controller import OptimizerController
from BTL.sample_function import LossFunctionType

controller = OptimizerController()

# Select loss function
controller.loss_function_idx = \
    controller.loss_function_types.index(LossFunctionType.HIMMELBLAU)

# Configure
controller.opt_start_x = -2.0
controller.opt_start_y = 2.0
controller.opt_learning_rate = 0.01
controller.max_epochs = 200

# Initialize
controller.activate_scene()

# Extract final losses
results = {}
for opt_type in controller.optimizer_order:
    controller.playback_time = float(controller.max_epochs - 1)
    controller._apply_playback_state()
    
    state = controller.display_state.get(opt_type, {})
    loss = state.get("loss", float('inf'))
    results[opt_type] = loss

# Print results
print("Final Loss Values:")
for opt_type, loss in sorted(results.items(), key=lambda x: x[1]):
    print(f"{opt_type.name:15s}: {loss:.6f}")

🚀 Optimization Algorithms Deep Dive

Overview

Algorithm Type Speed Best For
Batch GD First-order deterministic Slow Smooth, convex functions
SGD First-order stochastic Fast Large-scale learning
Mini-batch SGD First-order stochastic Medium Balance: stability vs speed
Momentum Accelerated Fast Escaping plateaus
Adam Adaptive Very Fast General purpose (recommended)

1. Batch Gradient Descent (BGD)

Update Rule

θ(t+1) = θ(t) - α * ∇f(θ(t))

Characteristics

  • Pros: Guaranteed convergence, smooth path, no noise
  • Cons: Slow convergence, can get stuck, high memory
  • Color: Red (#F24040)
  • Trajectory: Monotonically decreasing, smooth

2. Stochastic Gradient Descent (SGD)

Update Rule

θ(t+1) = θ(t) - α * ∇f_i(θ(t)) (where i = random sample)

Characteristics

  • Pros: Fast iterations, escapes local minima, low memory
  • Cons: Noisy updates, slower per-epoch convergence, requires tuning
  • Color: Orange (#FB6B16)
  • Trajectory: Noisy, zigzag pattern, generally downward

3. Mini-batch SGD

Update Rule

θ(t+1) = θ(t) - α * (1/|B|) * Σ ∇f_i(θ(t)) (where B = batch)

Characteristics

  • Pros: Balanced stability and speed, hardware-efficient
  • Cons: Batch size hyperparameter
  • Color: Blue (#1496FF)
  • Trajectory: Medium oscillation, faster than SGD
  • Default Batch Size: 16

4. Momentum

Update Rule

v(t+1) = γ * v(t) + ∇f(θ(t))
θ(t+1) = θ(t) - α * v(t+1)

Characteristics

  • Pros: Accelerated convergence, escapes plateaus, momentum
  • Cons: Can overshoot, oscillation risk
  • Color: Purple (#B33DFF)
  • Typical γ: 0.9 (provides 2-3x speedup)
  • Trajectory: Fast straight lines, potential oscillation

5. Adam (Adaptive Moment Estimation)

Update Rule

m(t) = β₁ * m(t-1) + (1 - β₁) * ∇f
v(t) = β₂ * v(t-1) + (1 - β₂) * (∇f)²
m̂(t) = m(t) / (1 - β₁^t)    [bias correction]
v̂(t) = v(t) / (1 - β₂^t)    [bias correction]
θ(t+1) = θ(t) - α * m̂(t) / (√v̂(t) + ε)

Characteristics

  • Pros: Adaptive learning rate, combines momentum + RMSprop, fast, robust
  • Cons: More parameters, slightly higher memory
  • Color: Green (#34E156)
  • Default LR: 0.001
  • Default β₁: 0.9
  • Default β₂: 0.999
  • Trajectory: Smooth, confident descent, graceful slowdown

Comparative Analysis

Algorithm Quadratic Booth Rosenbrock Overall
BGD ✓ Fast ✓ Fast ❌ Very Slow Medium
SGD ✓ Fast ✓ Fast ❌ Struggles Medium
Mini-batch ✓ Fast ✓ Fast ❌ Slow Medium
Momentum ✓✓ Very Fast ✓✓ Fast ✓ Fast High
Adam ✓✓ Very Fast ✓✓ Fast ✓✓ Fast Very High

📊 Loss Functions & Optimization Landscapes

Overview

Function Domain Global Min Type Difficulty
Quadratic 2D [-30, 30]² (0, 0) f=0 Trivial Very Easy
Booth [-30, 30]² (1, 3) f=0 Simple Easy
Beale [-4.5, 4.5]² (3, 0.5) f=0 Multimodal Medium
Himmelblau [-5, 5]² 4 minima f≈0 Multimodal Hard
Rosenbrock [-30, 30]² (1, 1) f=0 Valley Very Hard

1. Quadratic 2D (Baseline)

Definition

f(x, y) = x² + y²

Properties

  • Global minimum: (0, 0) with f = 0
  • Shape: Perfect paraboloid, spherical symmetry
  • Condition number: 1.0 (perfectly conditioned)
  • Gradient: Always points away from origin

Why Interesting

  • Trivial for optimization - all algorithms succeed equally
  • Baseline comparison for other functions
  • Perfect for theoretical analysis
  • Visual clarity - perfect bowl shape

2. Booth Function

Definition

f(x, y) = (x + 2y - 7)² + (2x + y - 5)²

Properties

  • Global minimum: (1, 3) with f = 0
  • Form: Sum of two linear functions, squared
  • Condition number: Low-moderate
  • Shape: Two perpendicular planes intersecting

Why Interesting

  • Simplest non-trivial function
  • Easy to understand geometrically
  • Good for teaching basics
  • All algorithms perform similarly

3. Beale Function

Definition

f(x,y) = (1.5 - x + xy)² + (2.25 - x + xy²)² + (2.625 - x + xy³)²

Properties

  • Global minimum: (3.0, 0.5) with f = 0
  • Characteristic: Multiple local minima, steep valleys
  • Condition number: High
  • Gradient: Can be misleading

Why Interesting

  • Multiple local minima create valleys
  • Tests optimizer robustness
  • Gradient can be misleading
  • Requires careful learning rates

4. Himmelblau Function

Definition

f(x, y) = (x² + y - 11)² + (x + y² - 7)²

Properties

  • Global minima: 4 global minima (all with f ≈ 0)
  • Locations: (3.0, 2.0), (-2.805, 3.131), (-3.779, -3.283), (3.584, -1.848)
  • Symmetry: Approximate 4-fold symmetry
  • Condition number: Very high (~1000)

Why Interesting

  • Multiple global minima - which will optimizers reach?
  • Deep valleys with different basins of attraction
  • No obvious pattern - tests algorithm behavior
  • Models real non-convex problems

5. Rosenbrock Function (The Classic)

Definition

f(x, y) = (1 - x)² + 100(y - x²)²

Properties

  • Global minimum: (1, 1) with f = 0
  • Characteristic: Very narrow, curved valley
  • Condition number: Extremely high (~1000+)
  • Created: 1960 by Howard H. Rosenbrock

Why Interesting

  • Classic benchmark - Gold standard for testing optimizers
  • Notoriously difficult - Even "easy" methods struggle
  • Non-convex valley - Curved path, misleading gradients
  • Tests acceleration - Which algorithms find the shortcut?

Why It Matters

The Rosenbrock function was created to:

  1. Have a single global minimum (no ambiguity)
  2. Be extremely difficult for optimization
  3. Reveal algorithmic shortcomings
  4. Require sophisticated solution methods

It remains the gold standard for testing and benchmarking optimization algorithms.

1. Flat Shading

Visual Characteristics

  • Constant color across triangle
  • Sharp edges between faces
  • Faceted, polygonal appearance
  • Hardware: Fastest possible

Mathematical Model

Color_triangle = ConstantColor

Use Cases

  • Wireframe preview
  • CAD visualization
  • Artistic rendering
  • Performance testing

Advantages

  • ✅ Fastest possible rendering
  • ✅ Clear geometry visualization
  • ✅ Useful for debugging
  • ✅ Minimal GPU utilization

Disadvantages

  • ❌ No lighting information
  • ❌ Unrealistic appearance
  • ❌ Can't see curvature
  • ❌ No specular highlights

2. Gouraud Shading

Visual Characteristics

  • Smooth color interpolation across triangle
  • Lighting computed at vertices
  • Colors interpolated in rasterization
  • Trade-off: speed vs quality

Mathematical Model

L_v = k_a + k_d(N · L) + k_s(R · V)^n

(Then barycentric interpolation in fragment)

Use Cases

  • Smooth geometries (spheres, cylinders)
  • Decent quality with good speed
  • Mobile rendering
  • Real-time applications

Advantages

  • ✅ Smooth appearance
  • ✅ Faster than Phong
  • ✅ Good balance for real-time
  • ✅ Suitable for curved surfaces

Disadvantages

  • ❌ Mach banding artifacts
  • ❌ Lighting anomalies on silhouettes
  • ❌ Can't show surface details
  • ❌ Specular highlights interpolate unnaturally

3. Phong Shading (Industry Standard)

Visual Characteristics

  • Per-fragment lighting calculation
  • Realistic specular highlights
  • Smooth appearance without banding
  • Industry standard quality

Phong Lighting Equation

L = k_a * I_a + Σᵢ [k_d * (N · Lᵢ) + k_s * (R · V)^n_s] * Iᵢ

Where:
  k_a, k_d, k_s = material coefficients
  N = surface normal
  L = light direction
  R = reflected light direction
  V = view direction
  n_s = shininess exponent
  I = light intensity

Components

  • Ambient: Constant background lighting, prevents complete darkness
  • Diffuse: Depends on light-surface angle, creates realism
  • Specular: Mirror-like reflections, depends on view angle

Shininess Effects

  • Shininess = 8: Large, soft highlight (rubber-like)
  • Shininess = 32: Medium highlight (plastic)
  • Shininess = 128: Sharp, small highlight (polished)
  • Shininess = 256: Very sharp (mirror-like)

Use Cases

  • Professional visualization
  • Game engines
  • CAD software
  • Scientific visualization

Advantages

  • ✅ Realistic appearance
  • ✅ No artifacts or banding
  • ✅ Industry standard
  • ✅ Good balance of quality/performance

Disadvantages

  • ❌ Slightly more expensive than Gouraud
  • ❌ Still uses Cook-Torrance approximation
  • ❌ Can't show fine surface details

4. Extended Phong (PhongEx)

Enhancements

  • Multiple material support
  • Metallic/roughness parameters
  • Emissive color channel
  • HDRI environment mapping
  • Normal map support (optional)

Visual Results

  • High-quality, production-ready rendering
  • Realistic material appearance
  • Dynamic reflections from environment

5. Color Interpolation

Visual Characteristics

  • Per-vertex color blend
  • Simple interpolation from vertex colors
  • No lighting calculations
  • Fast rendering

Use Cases

  • Colored point clouds
  • Custom gradient effects
  • Debugging with color coding
  • Heat maps and color visualization
🎨 Recommendation: Use Phong shading for most cases - it provides excellent quality without excessive performance cost.
